## What does this enhanced script do?

Think of this script as your personal XProtect configuration assistant that handles all the tedious, time-consuming tasks that nobody enjoys doing manually:

1. **Secure Antivirus Protection:** Automatically sets up recommended Windows Defender exclusions for XProtect - BUT ONLY ON STORAGE DRIVES. The script includes enhanced security by NEVER adding antivirus exceptions to your OS drive (C:), keeping your system fully protected while optimizing storage drives for video recording performance.

2. **Storage Drive Optimization:** Checks your storage drives for proper 64KB block size allocation and automatically disables indexing on storage drives (but keeps it on your OS drive, naturally). This is crucial for smooth 24/7 video recording.

3. **Windows Bloatware Removal:** Gets rid of all that unnecessary stuff Windows comes with - now updated for Windows 11 25H2! The script removes:
   - Standard bloatware (Cortana, Solitaire, Weather, etc.)
   - **25H2 newcomers: Disney+, TikTok, Spotify, Clipchamp** and other consumer apps
   - **Microsoft Teams (BOTH new app AND classic versions)**
   - **Outlook App (new Windows Outlook)**
   - **Office Hub/365 installer (the trial installer icon)**
   - OneDrive
   - **Copilot and Recall** are disabled (AI features have no place on a recording server!)
   - **Telemetry scheduled tasks** are disabled for privacy and performance

   **IMPORTANT:** Xbox apps are intentionally KEPT because removing them causes popup issues about missing screen recording components. This is a deliberate design choice based on extensive field testing.

4. **System Performance Tuning:** Sets your system to Never Sleep mode (critical for 24/7 recording!), disables unnecessary Windows services, enables LargeSystemCache for better disk I/O performance, optimizes Remote Desktop for management, and keeps the essential tools you actually need.

5. **NTP Time Server for Cameras (CRITICAL!):** Configures Windows Time Service as a reliable NTP (Network Time Protocol) server for your CCTV cameras. This is ABSOLUTELY CRITICAL for surveillance systems! The script:
   - Configures Windows Time Service as a reliable NTP server
   - Syncs the server time from Swedish NTP pool (se.pool.ntp.org)
   - Opens firewall (UDP port 123) for camera connections
   - Enables forensically accurate timestamps on all recordings
   - **v2.0 FIX:** Corrected NTP peer format and domain detection bugs from v1.9x

   **Why this matters:** Inaccurate timestamps can make your recordings legally inadmissible! Courts require precise, synchronized timestamps. This configuration ensures all your cameras have the exact same time, synchronized to atomic time sources.

6. **SSH Server Installation (NEW in v2.0!):** Optionally installs and configures OpenSSH Server on your XProtect machine. This allows secure remote management and troubleshooting without needing to use Remote Desktop for every task. Perfect for headless servers or when you need quick command-line access.

7. **NTP Recovery for Deployed Machines (NEW in v2.0!):** A dedicated recovery option that fixes NTP configuration on machines that were set up with the buggy v1.9x NTP configuration. Also available as a standalone script (NTP-Recovery.ps1) that you can run independently without the full toolkit.

8. **Windows Update Toggle:** Windows Update management has been separated from the Complete Setup and is now a standalone toggle. This gives you cleaner control - run the complete setup for system optimization, then decide about Windows Update separately.

9. **SNMP Monitoring Setup (OPTIONAL):** SNMP installation is completely optional. Most users don't need it! SNMP is only useful if you have network monitoring systems like PRTG, Nagios, or Zabbix. The script explains what SNMP is and when you need it, so you can make an informed decision.

10. **Milestone PowerShell Tools Installation:** Automatically downloads and installs the official Milestone PowerShell Tools from PowerShell Gallery. These powerful cmdlets let you manage XProtect systems programmatically.

11. **Auto-Detection:** The script automatically detects your Windows version (Windows 11, Server 2022, Server 2025, etc.) and adjusts its behavior accordingly. Uses modern Get-CimInstance calls instead of the deprecated Get-WmiObject for better compatibility with current Windows versions.

12. **Complete Automated Setup:** The crown jewel - available in two flavors! You'll still be asked whether you want optional SNMP installation.

13. **Comprehensive Logging:** Detailed logs with timestamps, plus automatic registry backups before any changes.

## FREQUENTLY ASKED QUESTIONS

**Q: What's new in v2.0 compared to v1.9?**
A: The big changes are: NTP bugs fixed (peer format + domain detection), SSH server option, 25H2 bloatware removal (Disney+, TikTok, Spotify, Clipchamp), Copilot/Recall disabled, telemetry tasks disabled, LargeSystemCache enabled, Windows Update separated from Complete Setup, Get-CimInstance replaces Get-WmiObject, and NTP Recovery for machines deployed with v1.9x. See the changelog above for full details.

**Q: I deployed machines with v1.9x and NTP isn't working. What do I do?**
A: Use menu option 7 (NTP Recovery) or run the standalone NTP-Recovery.ps1 script on the affected machines. Both fix the peer format and domain detection bugs from v1.9x. Run as administrator.

**Q: Why is NTP Time Server configuration so important?**
A: **Accurate time synchronization is CRITICAL for CCTV systems!** Here's why:
- **Legal Admissibility:** Courts can reject footage with inaccurate or unsynchronized timestamps
- **Forensic Analysis:** Investigators need precise timeline correlation across multiple cameras
- **Event Reconstruction:** You can't accurately reconstruct incidents if cameras have different times
- **Regulatory Compliance:** Many regulations require accurate, traceable time sources
- **Insurance Claims:** Claims are often denied if timestamps can't be verified as accurate

Without NTP, cameras drift - sometimes by minutes or even hours. This makes your expensive surveillance system nearly useless for its primary purpose: providing legally admissible evidence.

**Q: How do I configure my cameras to use the NTP server?**
A: After running the script:
1. Find the "Time" or "Date & Time" settings in each camera's web interface
2. Set NTP Server to your XProtect server's IP address
3. Enable NTP synchronization
4. Verify that time is syncing correctly

Most IP cameras sync time every few hours automatically once configured.

**Q: Can I use an external NTP server instead?**
A: You could, but there are advantages to using your XProtect server:
- **No internet required:** Cameras don't need internet access
- **Network isolation:** Cameras stay on your isolated CCTV network
- **Reliability:** No dependency on external services
- **Centralized control:** One time source for your entire system

The XProtect server itself syncs to Swedish atomic time sources, so it's just as accurate as using external NTP directly.

**Q: Why did you add SSH support?**
A: Many recording servers are headless or in locked server rooms. SSH gives you quick command-line access for troubleshooting, checking disk space, reviewing logs, or running PowerShell commands without needing a full Remote Desktop session. It's optional - only install it if your environment allows it.

**Q: Why is Windows Update no longer in the Complete Setup?**
A: Separation of concerns. The Complete Setup optimizes your system for XProtect. Windows Update management is a separate policy decision that varies by organization. Some want updates enabled for security, others want them disabled for stability during critical recording periods. Making it a separate toggle (option 6) makes the choice more deliberate and less likely to be accidentally set.

**Q: Why are Xbox apps kept? Can I still remove them?**
A: Xbox apps are intentionally kept because removing Xbox Game Bar causes Windows to display popup messages about "missing components needed for screen recording" on some systems. The popups are more annoying than the apps themselves. If you absolutely must remove them, use the individual options menu (option 3).

**Q: What is LargeSystemCache?**
A: It's a registry setting that tells Windows to allocate more RAM for disk caching operations. This is beneficial for video recording servers that do heavy sequential disk writes. The result is smoother disk I/O and better overall recording performance.

**Q: Why disable Copilot and Recall?**
A: A surveillance server should be a focused, purpose-built machine. Copilot uses resources and sends data to Microsoft. Recall takes periodic screenshots of your screen - on a surveillance system handling sensitive footage, that's a security and privacy concern. Neither has any business on a recording server.

**Q: What 25H2 bloatware does this remove?**
A: Windows 11 25H2 added consumer apps like Disney+, TikTok, Spotify, and Clipchamp. These are fine for personal computers but have zero purpose on a dedicated recording server. The script removes them along with the classic bloatware (Cortana, Solitaire, Weather, etc.).

**Q: Will this work with all camera brands?**
A: Yes! NTP is an industry standard protocol supported by virtually all IP cameras from all manufacturers (Axis, Hikvision, Dahua, Hanwha, etc.). If your camera has network connectivity, it almost certainly supports NTP.

**Q: Do I need SNMP?**
A: **Most users DON'T need SNMP!** You only need it if you have network monitoring tools like PRTG, Nagios, Zabbix, or SolarWinds. If you don't use any of these tools, you can safely skip SNMP installation. XProtect works perfectly without it.

**Q: Will this script work on Windows Server editions?**
A: Yes! The script auto-detects your Windows version and works on Windows 11, Windows Server 2022, and Windows Server 2025. Some bloatware apps may not be present on Server editions, and the script will simply skip them.

**Q: Can I run this script on a system that already has XProtect installed?**
A: Yes, but it's better to run it BEFORE installing XProtect. If XProtect is already installed, the script will still work, but you should test thoroughly afterward and may need to restart the XProtect services.

**Q: My cameras are still not syncing time. What should I check?**
A: Troubleshooting steps:
1. Verify Windows Time service is running: `Get-Service w32time`
2. Check NTP status: `w32tm /query /status`
3. Verify firewall allows UDP port 123: Check Windows Firewall rules
4. Ensure cameras can reach the server's IP address (ping test)
5. Check camera's NTP configuration (correct IP address entered?)
6. Some cameras need a reboot after changing NTP settings
7. **If deployed with v1.9x, run NTP Recovery first!**

**Q: Can I use this NTP server for other devices too?**
A: Yes! Any device that supports NTP can sync from this server - other computers, network equipment, access control systems, etc. That's one of the benefits of having a local NTP server.

**Q: Does this replace the need for a GPS time source?**
A: For most installations, yes. The server syncs to atomic time sources via internet, which is sufficient for legal requirements in most jurisdictions. GPS time sources are only needed for extremely critical applications or sites without reliable internet access.
